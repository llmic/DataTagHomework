
# 工业互联网年报AI标注辅助工具

## 一、项目概述
本工具是**上市公司年报工业互联网应用标注期中作业**专用自动化辅助系统，严格1:1对齐官方标注规则，实现从年报文本预处理、关键词筛选、上下文提取到最终结果生成的全流程自动化，大幅降低人工标注工作量，确保标注结果合规性。

- **任务目标**：从上市公司年报中筛选包含指定关键词的句子，标注企业是否**已实际落地应用**工业互联网相关技术（已应用=1，未应用/规划中/对外服务=0）
- **工具定位**：辅助人工标注，而非替代人工。所有AI标注结果均需人工复核确认
- **核心原则**：100%遵循官方标注规则，核心配置区无自定义修改，完全满足作业评分要求

## 二、快速开始
1. 安装Ollama并拉取推荐模型：`ollama pull deepseek-r1:8b`
2. 在项目根目录创建`data_source`文件夹，放入待标注的年报TXT文件
3. 安装Python依赖：`pip install -r requirements.txt`
4. 运行主程序：`python main.py`
5. 人工复核生成的总标注结果Excel文件，重命名后提交

## 三、环境准备
### 1. 基础软件
- **Python 3.8+**：官网下载安装，务必勾选"Add Python to PATH"选项
- **Ollama v0.1.30+**：本地大模型运行工具，下载地址：https://ollama.com/
  - Windows安装完成后，桌面右下角会出现Ollama图标，表示服务已自动启动
  - 若服务未启动，可在开始菜单搜索"Ollama"并点击启动，或执行命令：`ollama serve`

### 2. 拉取本地模型
打开命令提示符(CMD)或PowerShell，执行以下命令拉取推荐模型：
```bash
ollama pull deepseek-r1:8b
```
> 可选替代模型：`qwen2.5:7b`、`llama3:8b`、`gemma2:9b`，可在运行配置中修改模型名称

### 3. Python依赖库安装
打开CMD，进入项目根目录，执行以下命令安装所有依赖：
```bash
pip install regex pandas ollama
```
> 若安装失败，可尝试使用国内镜像源：
> ```bash
> pip install -i https://pypi.tuna.tsinghua.edu.cn/simple regex pandas ollama
> ```

## 五、详细使用流程
### 步骤1：准备年报数据
1. 在项目根目录创建名为`data_source`的文件夹
2. 将所有待标注的年报TXT文件放入该文件夹
3. 确保文件名遵循以下规范（否则无法正确解析股票代码和年份）：
   ```
   证券代码_年份_公司简称_年度报告类型.txt
   ```
   - 正确示例：`000001_2019_平安银行_2019年年度报告.txt`
   - 若文件名不规范，程序会自动将股票代码设为"000000"，年份设为"0000"

### 步骤2：配置运行参数（仅此处可修改）
打开`main.py`文件，找到最下方的**运行配置区**：
```python
# ====================== 运行配置（仅需修改此处）======================
REPORT_FOLDER = "./data_source"  # 年报文件存放路径
MODEL_NAME = "deepseek-r1:8b"    # 使用的本地模型名称
# ======================================================================
```
- 如需修改年报存放路径，调整`REPORT_FOLDER`变量
- 如需使用其他已拉取的模型，修改`MODEL_NAME`为对应的模型名称

### 步骤3：启动标注程序
1. 打开CMD，进入项目根目录
2. 执行命令：`python main.py`
3. 程序将自动完成以下操作：
   - 扫描并加载`data_source`文件夹内所有TXT文件
   - 对年报文本进行中文分句处理
   - 筛选包含官方指定关键词的句子
   - 提取每个句子的上下文（前后各5句）
   - 调用本地大模型进行标注推理
   - 每标注1句自动保存临时进度
   - 全部标注完成后生成汇总Excel和单文件Excel

### 步骤4：人工复核结果（必须执行）
> ⚠️ **重要警告**：AI标注存在约5%-10%的误差，**所有结果必须人工复核**，否则可能导致作业扣分
1. 打开生成的`班级+学号+姓名_总标注结果.xlsx`
2. 逐行检查"人工标注标签"列和"原因"列
3. 重点复核以下容易出错的情况：
   - 包含"拟、计划、将、探索、推进"等未来词汇的句子是否被误标为1
   - 描述对外销售产品、提供技术服务的句子是否被误标为1
   - 仅描述技术研发、专利申请的句子是否被误标为1
   - 主语模糊（如"行业"、"相关企业"）的句子是否被误标为1

### 步骤5：生成最终提交文件
1. 复核完成并修正所有错误标注后
2. 将总Excel文件重命名为：`班级+学号+姓名.xlsx`
3. 提交该文件给班长汇总即可

## 六、核心功能特性
1. **合规关键词筛选**
   - 严格使用官方指定的117个核心关键词，无遗漏、无新增
   - 自动去重并按长度降序排序，实现长词优先匹配，避免短词覆盖长词导致的匹配错误

2. **严格规则对齐**
   - 标注逻辑1:1照搬官方要求，无任何自定义规则
   - 精准区分"已应用/规划中/对外服务/模糊描述"等9种不同情况
   - 强制要求结合上下文判断，避免断章取义

3. **断点续标功能**
   - 每标注1句自动保存临时进度
   - 程序中断、电脑关机或断电后，重新运行会自动从上次中断处继续
   - 完全避免重复标注和数据丢失

4. **自动格式生成**
   - 生成的Excel文件完全匹配作业要求的格式
   - 包含：句子内容、来源文件、标注标签、股票代码、年份、标注原因
   - 同时生成总汇总表和单份年报结果表，满足作业不同提交要求

5. **批量处理能力**
   - 支持同时处理任意数量的年报文件
   - 自动按来源文件拆分结果，方便管理和核对

## 七、官方标注规则摘要
### 标注为1（已应用）的强制条件（必须同时满足）
1. 句子包含指定的工业互联网核心关键词
2. 描述的是**本企业自身**在生产、研发、管理等内部核心业务流程的改造优化
3. 包含明确的**已完成、已落地**的应用行为动词（如"部署了"、"采用了"、"实现了"）
4. 句子中绝对不包含"拟、计划、将、有望、探索、布局、推进"等表未来的词汇
5. 主语明确指向本企业，而非"行业、业内、相关企业"等模糊主语
6. 企业是工业互联网平台的**最终使用方**，而非平台建设方或服务提供方

### 标注为0（未应用）的条件（满足任意一条即判定为0）
1. 不包含指定的工业互联网核心关键词
2. 属于对外服务、销售、技术赋能、建设平台供第三方使用等乙方行为
3. 仅描述专利、技术研发、理论成果，未明确说明已实际落地应用
4. 属于战略规划、未来计划，包含上述未来时态词汇
5. 主语模糊，未明确指向本企业的实际落地行为
6. 仅描述通用数字化系统（如普通ERP、CRM），无工业互联网相关应用
7. 属于非工业领域的应用，与工业互联网场景无关
8. 无明确的已落地应用行为动词，无实际成效的模糊性描述
9. 企业仅为平台建设方、服务提供方，而非最终使用方

## 八、注意事项
1. **核心配置禁止修改**
   - 代码中"核心配置区"的关键词、标注规则、输出路径等严禁修改
   - 任何修改都可能导致标注结果不符合作业要求，影响最终成绩

2. **文件名规范**
   - 年报文件名必须包含股票代码和年份，否则无法正确解析
   - 建议使用从巨潮资讯网下载的年报原文，避免格式和编码问题

3. **模型使用说明**
   - 推荐使用`deepseek-r1:8b`，在标注准确率和推理速度上表现最佳
   - 如需使用在线模型，可自行修改`label_single_sentence`函数中的模型调用逻辑
   - 严格遵守作业要求，禁止使用在线AI进行批量标注，仅允许本地模型辅助

4. **人工复核要求**
   - 标注完成后必须进行全面人工复核
   - 复核时可参考"原因"列的AI推理过程，快速定位可能的错误
   - 建议每标注完3-5份年报就进行一次复核，避免最后集中处理

## 九、常见问题与异常处理
| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 无法读取年报文件，提示乱码 | 文件编码不兼容 | 联系学生助理更换编码正确的年报文件，或尝试用记事本另存为UTF-8编码 |
| 模型调用失败，提示"model not found" | 模型未拉取或名称拼写错误 | 执行`ollama list`查看已拉取的模型，确认模型名称正确，未拉取则执行`ollama pull deepseek-r1:8b` |
| Ollama服务未启动 | 安装后未自动启动服务 | 在开始菜单搜索"Ollama"并点击启动，或执行命令：`ollama serve` |
| 标注速度非常慢 | 电脑配置较低（尤其是内存不足） | 可尝试使用更小的模型如`qwen2:7b`，关闭其他占用内存的程序，耐心等待 |
| 生成的Excel文件打开乱码 | 编码问题 | 使用Microsoft Excel打开，不要使用WPS；若仍乱码，可尝试用记事本打开后另存为UTF-8编码 |
| 程序中断后重新运行，从头开始标注 | 临时进度文件丢失或损坏 | 检查项目根目录是否存在`临时标注进度.csv`文件，若不存在则只能重新标注 |
| 依赖安装失败 | 网络问题或Python版本不兼容 | 使用国内清华镜像源安装，或升级Python到3.8以上版本 |
| 提示"权限不足" | 文件夹权限问题 | 以管理员身份运行CMD，或将项目文件夹移动到非系统盘（如D盘） |
